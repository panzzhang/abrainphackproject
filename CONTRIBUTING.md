# Contributing to The Brainphack Project

Thank you for your interest in contributing to The Brainphack Project! We welcome contributions from researchers, developers, and anyone interested in improving research transparency and reproducibility.

## How to Add Yourself as a Contributor

If you've contributed to this project, please add yourself to the contributors list by following these steps:

### 1. Fork and Clone the Repository (if applicable)

If you're working on your own fork:
```bash
git clone https://github.com/YOUR_USERNAME/Brainphack25.git
cd Brainphack25
```

### 2. Add Your Information to the Contributors List

Edit this file (CONTRIBUTING.md) and add your details to the Contributors section below in alphabetical order by last name:

```markdown
- **Your Name** - [Role/Contribution] - [GitHub](https://github.com/yourusername) | [Email](mailto:your.email@example.com)
```

For example:
```markdown
- **Jane Doe** - Literature collection and pipeline coding - [GitHub](https://github.com/janedoe) | [Email](mailto:jane@example.com)
```

### 3. Commit Your Changes

```bash
git add CONTRIBUTING.md
git commit -m "Add [Your Name] to contributors"
```

### 4. Push and Create a Pull Request (if applicable)

```bash
git push origin main
```

Then create a pull request on GitHub.

## Types of Contributions

We welcome various types of contributions, including but not limited to:

- **Literature collection**: Gathering and curating papers for analysis
- **Metric development**: Refining the phail metric definition and computation
- **Pipeline coding**: Implementing analysis pipelines for selected papers
- **Library development**: Building the open-source phail computation library
- **Visualization**: Creating plots and summary statistics
- **Automation**: Developing LLM-based extraction tools
- **Documentation**: Improving project documentation and tutorials
- **Manuscript writing**: Contributing to the research report/manuscript
- **Code review**: Reviewing pull requests and suggesting improvements
- **Bug reports**: Identifying and reporting issues

## Code of Conduct

We are committed to providing a welcoming and inclusive environment for all participants. Please:

- Be respectful and considerate in all communications
- Welcome newcomers and help them get started
- Accept constructive criticism gracefully
- Focus on what is best for the project and the community

## Getting Started

1. Check out the project goals and milestones in the [README.md](README.md)
2. Join the project communication channel (if available)
3. Look for issues labeled "good first issue" or "help wanted"
4. Reach out to the team leads for your area of interest

## Questions?

If you have questions about contributing, please open an issue or contact the project maintainers.
